+## Research Context (optional)

+Goal:
+- Сделать отображение погоды компактнее и понятнее: сократить текст описания, убрать лишние подписи у прогноза и устранить наложение цифр температуры/влажности.
+
+Constraints:
+- Сохранять читаемость на небольшом TFT‑экране 240x240.
+- Не увеличивать расход памяти и флеша заметно; переиспользовать существующие переменные и функции.
+- Следовать архитектуре: изменения только в UI‑слое и ближайшей логике, без изменения протокола получения данных.
+
+Decisions:
+- Сократить длинные текстовые описания погоды до коротких форм («Переменная облачность» → «перем. обл.» и т.п.).
+- Подпись под иконкой погоды рисовать только для текущего дня; для прогноза на завтра/послезавтра выводить только иконку и числовые значения.
+- Перед отрисовкой температуры и влажности явно очищать соответствующие области, а не пытаться «перерисовать поверх».
+
+Open questions:
+- Нужны ли дополнительные сокращения для других длинных описаний (можно добавить по мере выявления).
+
+## Commit Plan
+- **Commit 1**: "feat: compact current weather description and labels"
+- **Commit 2**: "fix: clear temperature and humidity areas before redraw"
+
+## Tasks
+
+### Phase 1: Setup
+- [ ] Найти в `main.cpp` логику отрисовки текущей погоды (иконка + подпись) и прогнозов на 2 дня, зафиксировать используемые координаты и области очистки.
+- [ ] Найти или добавить (при необходимости) функцию сопоставления `weather.icon` → строковое описание, чтобы аккуратно внедрить сокращённые подписи.
+
+### Phase 2: Core Implementation
+- [ ] Реализовать таблицу сокращённых подписей для длинных описаний погоды (например, «Переменная облачность» → «перем. обл.», «Небольшой дождь» → «небольш. дождь» и т.д.) с учётом ограничения по ширине текста; при этом для текущего дня по‑прежнему использовать одну строку подписи под иконкой.
+- [ ] Обновить расчёт ширины текста и центрирование иконки/подписи текущего дня так, чтобы опираться на максимально возможную строку сокращённого описания (чтобы ничего не срезалось по краям).
+- [ ] Изменить отрисовку прогноза на завтра/послезавтра так, чтобы:
+  - не выводить текстовые подписи‑описания под иконками;
+  - показывать только иконку и числовые значения температур/влажности;
+  - корректно очищать области прогноза при изменении данных (одной или двумя `fillRect` на каждую половину экрана, без затрагивания текущего дня).
+- [ ] Добавить явное стирание прошлой температуры:
+  - очистить прямоугольную область, в которой рисуются `tempStr`, символ "°" и "C" перед рисованием новой температуры;
+  - подобрать размеры области так, чтобы не задевать соседние элементы (дату сверху и иконку прогноза снизу).
+- [ ] Добавить явное стирание прошлой влажности:
+  - очистить прямоугольную область, в которой рисуются число, пробел и "%";
+  - убедиться, что очистка не задевает зону прогноза ниже.
+

+### Phase 3: Integration
+- [ ] Проверить на реальном устройстве (или в логах координат) отсутствие наложения текста: несколько циклов обновления погоды/прогноза, смена значений температуры/влажности.
+- [ ] При необходимости подкорректировать координаты/размеры очищаемых блоков (ширину/высоту `fillRect`), чтобы нигде не оставались артефакты старых значений.
+- [ ] Обновить комментарии в `main.cpp` рядом с зонами отрисовки погоды (кратко описать области экрана и логику очистки), не меняя общий стиль проекта.
