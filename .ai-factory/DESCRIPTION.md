# Project: WiFi Clock (ESP8266)

## Overview
WiFi‑управляемые часы на ESP8266 с TFT‑дисплеем, показывающие текущее время и дату, данные погоды и статус воздушной тревоги, а также имеющие веб‑интерфейс для настройки параметров без перепрошивки.

## Core Features
- Отображение текущего времени, даты и дня недели на TFT‑экране с крупными шрифтами.
- Периодическая синхронизация времени по NTP с настраиваемым сервером и часовым поясом.
- Получение текущей погоды и краткого прогноза с OpenWeatherMap (город и API‑ключ настраиваются).
- Отображение температуры, влажности и погодного статуса на экране.
- Получение и индикация статуса воздушной тревоги для выбранного региона.
- Веб‑интерфейс на ESP8266 для настройки WiFi, NTP и параметров погоды.
- Хранение пользовательских настроек во внутренней файловой системе LittleFS.
- Подробный лог в Serial‑порт для отладки (скорость 115200 бод).

## Tech Stack
- **Language:** C++ (Arduino)
- **Framework:** Arduino (ESP8266, PlatformIO)
- **Board:** ESP8266 (esp12e)
- **Display:** TFT_eSPI (настройки в `lib/User_Setup.h`)
- **Filesystem:** LittleFS
- **Networking:** ESP8266WiFi, ESP8266WebServer, WiFiClientSecure
- **Data Format:** JSON (ArduinoJson)
- **Integrations:**
  - OpenWeatherMap API для данных погоды и прогноза
  - Внешний HTTP‑API для статуса воздушной тревоги

## Architecture Notes
- Одно основное приложение в `src/main.cpp` с функциями `setup()` и `loop()`.
- Логика инициализации железа (дисплей, WiFi, файловая система) выполняется в `setup()`.
- Для отображения времени и цифр используются отдельные шрифтовые/графические ресурсы в файлах `Robot20.h`, `Robot35.h`, `Robot60.h`.
- Конфигурация (WiFi, NTP, погода) хранится во внутренней файловой системе LittleFS и может изменяться через встроенный веб‑сервер.
- Регулярные задачи (обновление времени, погоды, статуса тревоги, перерисовка экрана) выполняются в `loop()` с таймерами на основе `millis()`.
- Для экономии ресурсов применён кэш последнего отображённого состояния (время, дата, погода, WiFi‑статус) и перерисовываются только изменившиеся области.

## Non-Functional Requirements
- **Логирование:** человекочитаемые сообщения в Serial‑порт, включая этапы загрузки, подключения к WiFi и ошибки сетевых запросов.
- **Надёжность сети:** простая повторная попытка подключения к WiFi при старте, защита от зависания при недоступности NTP или API.
- **Хранение конфигурации:** устойчивое к перезапуску хранение пользовательских настроек в LittleFS.
- **UX:** крупный шрифт часов, читаемое отображение погоды и статуса тревоги, возможность изменить ключевые параметры через веб‑интерфейс.
- **Безопасность:** использование HTTPS (WiFiClientSecure) для внешних API там, где это возможно; хранение ключа OpenWeatherMap в настройках устройства.
