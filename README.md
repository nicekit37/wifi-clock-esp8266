# WiFi Clock (ESP8266, TFT 240x240)

<img width="1024" height="769" alt="image" src="https://github.com/user-attachments/assets/48d485a8-3d0b-4c26-abfc-3ac5e11b444b" />


Проект прошивки для настенных/настольных часов на базе **ESP8266 (ESP-12E)** с цветным TFT‑дисплеем 240×240.  
Часы показывают текущее время и дату, погоду и прогноз, статус воздушной тревоги, а также имеют веб‑интерфейс для настройки без перепрошивки.

## Возможности

- **Часы и дата**
  - Крупное цифровое время (часы/минуты) с мигающим двоеточием.
  - Отображение дня недели и текущей даты в удобном виде.
  - Секунды отдельным шрифтом.

- **Синхронизация времени (NTP)**
  - NTP‑сервер и часовой пояс настраиваются.
  - Неблокирующая инициализация NTP при подключении к Wi‑Fi.
  - Корректная работа при временной потере интернета.

- **Погода**
  - Получение текущей погоды и краткого прогноза с **OpenWeatherMap**.
  - Отображение:
    - температуры и влажности крупными цифрами,
    - иконки текущей погоды,
    - короткого текстового описания (например, `Перем обл`).
  - Прогноз на 2 дня: иконка, min/max температура и влажность по дням.
  - Обновление погоды раз в 10 минут.

- **Статус воздушной тревоги**
  - Запрос к публичному API `ubilling.net.ua/aerialalerts`.
  - Определение тревоги по Одесской области (по нескольким вариантам написания).
  - Отдельная иконка тревоги на экране:
    - зелёная — всё спокойно,
    - мигающая/красная — тревога активна.

- **Wi‑Fi и веб‑интерфейс**
  - Автоматическое подключение к заданной сети Wi‑Fi.
  - **Автопереподключение**, если сеть появилась позже (через 1–2 минуты и т.п.).
  - Встроенный веб‑сервер на ESP8266:
    - просмотр текущих параметров,
    - изменение Wi‑Fi, NTP, настроек погоды и др.
  - Настройки сохраняются во внутренней файловой системе **LittleFS**.

- **Отрисовка и производительность**
  - Используются пользовательские шрифты `Robot20/35/60` для времени/даты/погоды.
  - Экран перерисовывается **частично**:
    - кешируются последние отображённые значения (время, дата, погода, Wi‑Fi),
    - при изменении значения стирается только нужный прямоугольник, чтобы не было наложений.
  - Отдельные функции для отрисовки времени, даты, погоды, тревоги.

## Аппаратные требования

- **Контроллер**: ESP8266 (плата `esp12e` / NodeMCU v2/v3).
- **Дисплей**: TFT 240×240, библиотека **TFT_eSPI**.
  - Пины и параметры дисплея настраиваются в `lib/User_Setup.h`.
- **Память**: рекомендуемый layout `4M (2M SPIFFS)` (см. `platformio.ini`).
- **Подключение к ПК**: USB‑UART (загрузчик от PlatformIO).

## Стек и инструменты

- **Язык / платформа**: C++ / Arduino (ESP8266, PlatformIO).
- **Библиотеки:**
  - `TFT_eSPI` — работа с TFT‑дисплеем.
  - `ESP8266WebServer` — встроенный веб‑сервер.
  - `ArduinoJson` — парсинг JSON (погода, тревоги).
  - `LittleFS` — файловая система во флеше для настроек и шрифтов.
- **Сборка**: [PlatformIO](https://platformio.org/).

## Установка и сборка (PlatformIO)

1. Установить [VS Code + PlatformIO](https://platformio.org/install).
2. Клонировать репозиторий:
   ```bash
   git clone https://github.com/<your-user>/wifi-clock-esp8266.git
   cd wifi-clock-esp8266
   ```
3. Открыть папку проекта в VS Code / PlatformIO.
4. Выбрать среду `esp12e` (см. `platformio.ini`).
5. Подключить плату ESP8266, указать нужный `COM`‑порт в `platformio.ini` (если отличается).
6. Собрать и прошить:
   ```bash
   pio run --target upload
   pio device monitor
   ```

## Конфигурация секретов

Чувствительные данные (Wi‑Fi, API‑ключи, URL‑ы) **не хранятся в репозитории**.  
Они задаются в локальном файле `include/secrets.h`, который игнорируется git’ом.

### Шаг 1. Создать `secrets.h`

Скопировать пример:

```text
include/secrets_example.h → include/secrets.h
```

Отредактировать `include/secrets.h` под свои данные:

```cpp
// Wi-Fi по умолчанию
#define WIFI_SSID        "MyWiFi"
#define WIFI_PASSWORD    "MySecretPass"

// NTP сервер
#define NTP_SERVER       "pool.ntp.org"

// OpenWeatherMap
#define WEATHER_API_KEY  "YOUR_OPENWEATHER_API_KEY"
#define WEATHER_CITY     "YourCityName"

// API воздушной тревоги
#define ALERT_API_URL    "http://ubilling.net.ua/aerialalerts/"

// OTA метаданные (JSON c версией и ссылкой на firmware.bin)
#define OTA_META_URL     "https://raw.githubusercontent.com/<user>/wifi-clock-esp8266/main/ota/latest.json"
```

Файл `include/secrets.h` уже добавлен в `.gitignore` и **не попадёт** на GitHub.

### Шаг 2. Настройки через веб‑интерфейс

После прошивки:

1. Устройство подключится к заданной Wi‑Fi сети (из `secrets.h`).
2. По IP‑адресу ESP (см. Serial Monitor) доступен веб‑интерфейс.
3. Через веб‑страницу можно изменить:
   - SSID/пароль Wi‑Fi,
   - NTP‑сервер и часовой пояс,
   - API‑ключ и город погоды и др.
4. Настройки сохраняются в LittleFS и используются при следующей загрузке.

## Автопереподключение к Wi‑Fi

В `loop()` вызывается функция `ensureWiFiConnected()`:

- Если `WiFi.status() == WL_CONNECTED`:
  - При первом успешном подключении:
    - вызывается `setupNTP()`,
    - сразу запрашивается погода (`updateWeather()`),
    - сбрасывается таймер проверки тревог.
- Если соединение потеряно:
  - раз в `wifiReconnectInterval` (30 c) отправляется новая попытка `WiFi.begin(...)` без длинных блокировок;
  - как только сеть появится — всё автоматически восстановится.

## Версионирование и релизы

- Текущая версия прошивки задаётся в `include/main.h`:

  ```cpp
  #define FIRMWARE_VERSION "1.0.2"
  ```

- Для автоматизации релизов есть скрипт `tools/release.py`:
  - читает `FIRMWARE_VERSION`,
  - создаёт/обновляет `CHANGELOG.md`,
  - делает коммит, git‑тег `vX.Y.Z` и пушит на GitHub.

Пример:

```bash
python tools/release.py "Wi-Fi автоподключение и улучшение отображения погоды"
```

После этого в репозитории появится тег `v1.0.2`, а в `CHANGELOG.md` — соответствующая запись.

## OTA-обновление (автоматически)

Прошивка умеет автоматически проверять и устанавливать обновления по Wi‑Fi:

- Проверка выполняется **сразу после подключения к Wi‑Fi**, а затем **раз в час**.
- Во время обновления на дисплее показывается экран `OTA UPDATE` с прогресс‑баром.

### Как это работает

Устройство читает файл `ota/latest.json` по адресу `OTA_META_URL`. Формат файла:

```json
{
  "version": "1.0.5",
  "url": "https://github.com/<user>/wifi-clock-esp8266/releases/download/v1.0.5/firmware.bin"
}
```

Если `version` отличается от `FIRMWARE_VERSION`, устройство скачивает `url` и прошивается через OTA.

### Автоматическая сборка и публикация firmware.bin (GitHub Actions)

В репозитории настроен workflow `.github/workflows/release.yml`:

- при пуше тега `vX.Y.Z` GitHub Actions:
  - собирает прошивку PlatformIO (`esp12e`),
  - создаёт GitHub Release и прикрепляет `firmware.bin`,
  - обновляет `ota/latest.json` на новую версию и ссылку на `firmware.bin`.

То есть для выпуска новой версии достаточно:

1) Обновить `FIRMWARE_VERSION` (например, на `1.0.5`) и запушить изменения в `main`.

2) Создать и запушить тег:

```bash
git tag -a v1.0.5 -m "WiFi Clock 1.0.5"
git push origin v1.0.5
```

## Структура проекта

Основные файлы:

- `platformio.ini` — конфигурация PlatformIO (плата, библиотеки, порты).
- `src/main.cpp` — основная логика прошивки (`setup()`, `loop()`, отрисовка, сеть, погода, тревоги).
- `include/main.h` — объявления глобальных переменных и функций.
- `include/secrets_example.h` — пример конфигурации секретов.
- `include/secrets.h` — **локальные секреты (в git не коммитится)**.
- `lib/User_Setup.h` — настройки TFT_eSPI (тип дисплея, пины).
- `src/Robot20.h`, `Robot35.h`, `Robot60.h` — пользовательские шрифты.

Также в проекте есть AI‑контекст файлы (например, `AGENTS.md`, `.ai-factory/DESCRIPTION.md`), которые помогают AI‑ассистенту ориентироваться в коде; они не обязательны для сборки, но полезны при дальнейшем развитии проекта.
