# Решение проблемы с отображением символов

## Проблема
Символы "МІЗ-МА" отображаются как прямоугольники, "ТС" видна только буква С.

## Возможные причины
1. **Шрифт Calibri не содержит кириллические символы**
2. **Неправильные Unicode коды**
3. **Проблемы с кодировкой**

## Решения

### Решение 1: Использовать системный шрифт Arial
Запустите скрипт **Create_arial_minimal.pde**:
- Arial имеет лучшую поддержку кириллицы
- Использует системный шрифт Windows

### Решение 2: Использовать системный шрифт
Запустите скрипт **Create_system_font_minimal.pde**:
- Использует системный шрифт с поддержкой кириллицы
- Автоматически выбирает лучший доступный шрифт

## Как проверить проблему

1. **Запустите любой из скриптов**
2. **Посмотрите в окне предварительного просмотра**:
   - Если символы отображаются как прямоугольники - шрифт не содержит эти символы
   - Если символы отображаются правильно - проблема в другом месте

## Альтернативные решения

### Вариант 1: Изменить текст в коде
Если проблема в шрифте, можно изменить текст:

```cpp
// Вместо "МІЗ-МА" использовать:
tft.println("MIZ-MA");  // Латинскими буквами

// Вместо "ТС" использовать:
tft.println("TS");      // Латинскими буквами
```

### Вариант 2: Использовать другой шрифт
Попробуйте использовать встроенные шрифты TFT_eSPI:

```cpp
// Вместо кастомного шрифта:
tft.setTextFont(4);  // Встроенный шрифт
tft.println("МІЗ-МА");
```

### Вариант 3: Создать шрифт с латинскими аналогами
Создать шрифт только с латинскими буквами:

```cpp
// Unicode коды для латинских аналогов:
// M = 0x004D, I = 0x0049, Z = 0x005A, A = 0x0041
// T = 0x0054, S = 0x0053
```

## Проверка Unicode кодов

Убедитесь, что используете правильные Unicode коды:

- М = 0x041C (кириллическая М)
- І = 0x0406 (украинская І)
- З = 0x0417 (кириллическая З)
- А = 0x0410 (кириллическая А)
- Т = 0x0422 (кириллическая Т)
- С = 0x0421 (кириллическая С)

## Рекомендации

1. **Сначала попробуйте Arial** - у него лучшая поддержка кириллицы
2. **Проверьте предварительный просмотр** в Processing IDE
3. **Если не помогает** - используйте латинские аналоги
4. **Проверьте кодировку файлов** - должны быть в UTF-8

## Тестирование

После создания нового шрифта протестируйте:

```cpp
tft.loadFont(arial40);  // или system40
tft.print("МІЗ-МА");
tft.print("ТС");
tft.print("УВАГА!");
tft.unloadFont();
```

Если символы все еще не отображаются, проблема может быть в:
- Кодировке исходного кода
- Настройках компилятора
- Версии библиотеки TFT_eSPI
